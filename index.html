    #3 SCRUM - Why it's called framework and not method?

    While a method should be followed step by step, a framework is meant to be implemented to an extend that actually works for each company's situation. That means that the framework, e.g. Scrum can and should be adapted to the organization's specific situation.

    For example, the Scrum guide says that there should be one Scrum team per project but if the company only consists of a 5-members-team it might still be possible that the team is working on different projects at the same time. This comes along with some challenges, e.g. the Scrum metrics of the individual projects should not be mixed. Also, the required workload of each project per sprint can vary. Furthermore, the product owner probably has to handle multiple customers at the same time.
